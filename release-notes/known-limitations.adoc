= AI Data Engine의 알려진 제한 사항
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
알려진 제한 사항은 이 제품 릴리스에서 지원되지 않거나 제대로 상호 운용되지 않는 플랫폼, 장치 또는 기능을 나타냅니다. 이러한 제한 사항을 주의 깊게 검토하십시오.



== 타사 서버 배포 환경에서 AIDE 소프트웨어의 알려진 제한 사항

이러한 제한 사항은 고객이 관리하는 하드웨어를 사용하는 타사 서버에 AIDE 소프트웨어를 배포하는 경우에 특별히 적용됩니다.



=== 배포 모델 제한 사항

* **3노드 클러스터만 해당**
+
타사 서버에 AIDE 소프트웨어를 배포하려면 정확히 3개의 노드가 필요합니다. 3개 미만의 노드로는 배포할 수 없으며, 3개 이상의 노드로 확장할 수도 없습니다.

* **동일한 하드웨어 필요**
+
타사 서버의 AIDE 소프트웨어 클러스터에 있는 모든 노드는 동일한 하드웨어 사양(동일한 CPU 모델, 메모리, 네트워크 구성 및 서버 모델)을 가져야 합니다. 혼합 하드웨어 구성은 지원되지 않습니다.

* **RHEL 9.7만 해당**
+
타사 서버에 AIDE 소프트웨어를 배포할 경우 Red Hat Enterprise Linux(RHEL) 9.7 LTS만 지원합니다. 다른 Linux 배포판은 지원되지 않습니다.

* **단일 사이트 배포만 해당**
+
타사 서버의 AIDE 소프트웨어는 단일 사이트 클러스터 배포만 지원합니다. 다중 사이트, 확장 클러스터 및 지리적으로 분산된 클러스터 토폴로지는 지원되지 않습니다. 타사 서버의 AIDE 소프트웨어 클러스터에 있는 세 개의 노드는 모두 지연 시간이 짧은 상호 연결을 통해 동일한 물리적 사이트에 있어야 합니다.

+

NOTE: 원격 사이트에 저장된 데이터의 메타데이터를 카탈로그화하려면 ONTAP 클러스터 피어링을 사용하여 원격 ONTAP 클러스터를 AIDE 클러스터에 연결하십시오.





=== 기능 범위 제한 사항

* **GPU 종속 서비스 없음**
+
타사 서버의 이번 AIDE 소프트웨어 릴리스는 벡터화, OCR(광학 문자 인식) 및 보강 기능을 포함한 GPU 기반 서비스를 지원하지 않습니다.

* **데이터 수집 없음**
+
데이터 컬렉션을 생성하려면 GPU 리소스를 포함한 AIDE 전체 배포가 필요합니다. 타사 서버에 설치된 AIDE 소프트웨어는 메타데이터 카탈로그 생성 기능만 제공합니다.

* **RAG 엔드포인트 없음**
+
의미 검색을 위한 RAG(Retrieval-Augmented Generation) 엔드포인트를 사용하려면 전체 AIDE를 배포해야 합니다. 타사 서버에 설치된 AIDE 소프트웨어는 벡터 임베딩이나 의미 검색 기능을 지원하지 않습니다.

* **검색 시점에 가드레일 정책이 시행되지 않음**
+
검색 시점 정보 삭제 및 액세스 제어를 위한 가드레일 정책에는 벡터화 기능을 갖춘 완전한 AIDE가 필요합니다.





=== 업그레이드 경로

* **롤백 동작은 릴리스별로 다릅니다.**
+
대상 릴리스 지침에서 지원하는 경우 이전에 설치된 AIDE 소프트웨어 번들 버전으로 롤백할 수 있습니다. 해당 릴리스에 대한 문서화된 롤백 절차를 따르십시오.





=== 운영 체제 및 소프트웨어 관리

* **추가 소프트웨어는 지원되지 않습니다**
+
타사 서버 노드에서 AIDE 소프트웨어 이외의 애플리케이션이나 서비스를 실행하는 것은 지원되지 않습니다.





=== ONTAP 및 스토리지 요구 사항

* **ONTAP 9.18.1 이상 필요**
+
타사 서버의 AIDE 소프트웨어에는 ONTAP 9.18.1 이상이 필요합니다. 이전 ONTAP 버전은 지원되지 않습니다.

* **AFX 스토리지 필요**
+
타사 서버의 AIDE 소프트웨어는 NetApp AFX 스토리지 시스템에 연결해야 합니다. 다른 ONTAP 플랫폼은 지원되지 않습니다.





== AIDE 9.18.1U0/U1 초기 릴리스의 알려진 제한 사항

이러한 제한 사항은 AIDE 9.18.1U0/U1의 Metadata Engine, 데이터 컴퓨팅 노드 및 관리 워크플로에 적용됩니다.



=== 컴퓨팅 노드 요구사항 및 관리

* **최소 NetApp DCN 요구 사항**
+
AIDE 클러스터는 Metadata Engine 기능을 사용하기 위해 정확히 3개의 NetApp DCN 노드가 필요합니다. NetApp DCN 노드가 3개 미만인 클러스터에서는 Metadata Engine 기능을 활성화할 수 없습니다.

* **NetApp Console을 사용한 NetApp DCN 노드 추가는 지원되지 않습니다**
+
NetApp DCN 노드 업그레이드 및 추가는 NetApp Console이 아닌 ONTAP System Manager를 사용하여 수행해야 합니다.





=== 지원되는 데이터 소스

* **ONTAP S3 버킷 또는 StorageGRID를 데이터 소스로 지원하지 않습니다**
+
워크스페이스 및 메타데이터 카탈로그의 데이터 소스로는 ONTAP 볼륨(로컬 또는 원격)만 지원됩니다. 이번 릴리스에서는 ONTAP S3 버킷 및 StorageGRID 객체를 워크스페이스에 추가할 수 없으며 Metadata Engine에서 인덱싱하지 않습니다.

* **FlexCache 볼륨을 사용한 워크스페이스 생성은 지원되지 않습니다**
+
FlexCache 볼륨은 워크스페이스의 데이터 소스로 추가할 수 없습니다.





=== 소프트웨어 업데이트 및 되돌리기 제한 사항

* **NetApp DCN 노드에 대한 수동 소프트웨어 업데이트만 해당**
+
AIDE 9.18.1에서는 NetApp DCN 클러스터의 자동 소프트웨어 업데이트가 지원되지 않습니다. NetApp DCN 소프트웨어는 로컬 클라이언트에서 이미지를 업로드하는 방식으로만 업데이트할 수 있습니다. 외부 서버(HTTP/FTP)에서 이미지를 다운로드하는 것은 지원되지 않습니다.

* **NetApp DCN 클러스터 소프트웨어는 이전 버전으로 되돌릴 수 없습니다.**
+
NetApp DCN 클러스터 소프트웨어는 이전 버전으로 되돌릴 수 없습니다. 최신 버전으로만 업그레이드할 수 있습니다.

* **AFX 스토리지 클러스터의 ONTAP 되돌리기 없음**
+
AFX 스토리지 클러스터는 이전 ONTAP 버전으로 되돌릴 수 없습니다. 최신 버전으로만 업그레이드할 수 있습니다.





=== 워크스페이스 수명 주기 및 액세스 구성

* **워크스페이스에 대한 소프트 삭제 또는 복원 없음**
+
워크스페이스를 삭제하면 영구적으로 삭제됩니다. 삭제된 워크스페이스는 복원할 수 없습니다.

* **초기 클러스터 설정 시 OIDC 구성은 지원되지 않습니다**
+
OIDC/OAuth 구성은 ONTAP System Manager를 사용하여 클러스터를 생성한 후에 수행해야 합니다.
