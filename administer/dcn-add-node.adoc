= AI Data Engine 클러스터에 데이터 컴퓨팅 노드 추가
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
새로운 AI Data Engine(AIDE) 클러스터를 생성하거나 기존 클러스터를 확장하는 과정에서 데이터 컴퓨팅 노드를 추가할 수 있습니다. 워크플로는 ONTAP System Manager를 사용하여 노드를 검색하고 구성하는 것으로 구성됩니다.



== 노드 추가 준비

데이터 연산 노드를 추가할 때 다음 사항을 고려하십시오.

.최소 데이터 컴퓨팅 노드 수
새로운 AIDE 클러스터에는 정확히 3개의 사용 가능한 데이터 컴퓨팅 노드가 필요합니다.

.하드웨어 설치 및 주소 지정 가능성
다음 필수 조건을 충족하는지 확인하십시오.

* 새로운 데이터 컴퓨팅 하드웨어가 랙에 설치되고 전원이 켜졌으며 클러스터 스위치에 케이블로 연결되었습니다.
* 데이터 컴퓨팅 노드에서 ONTAP 백엔드 서브넷까지 사용할 수 있는 IP 주소 공간 범위가 있습니다.
* ONTAP 클러스터가 초기화되었으며 클러스터 관리 LIF에서 연결할 수 있습니다.


.System Manager 자격 증명
AIDE 클러스터 생성 또는 확장 작업을 수행하려면 _스토리지 관리자_ 권한이 필요합니다.

.소프트웨어 호환성
데이터 컴퓨팅 하드웨어 및 소프트웨어 버전이 ONTAP 클러스터와 호환되는지 확인하려면 다음 문서를 검토하십시오.

* link:../administer/upgrades-aide-summary.html["AIDE 소프트웨어는 ONTAP 버전과 호환됩니다"].


노드 추가 작업 중에 System Manager는 새 노드가 다음과 호환되는 소프트웨어 버전을 실행하고 있는지 확인합니다.

* 이것이 처음으로 추가된 데이터 컴퓨팅 노드인 경우 ONTAP 클러스터의 유효 버전입니다.
* 데이터 컴퓨팅 노드가 이미 있는 경우 기존 데이터 컴퓨팅 클러스터 버전입니다.


노드가 호환되지 않는 경우:

* *추가* 대화 상자에서 영향을 받는 데이터 컴퓨팅 노드 옆에 오류가 표시됩니다.
* 먼저 데이터 컴퓨팅 노드 소프트웨어(또는 해당되는 경우 ONTAP)를 호환되는 버전으로 업데이트해야 합니다.




== 데이터 컴퓨팅 노드 추가

새로운 AIDE 클러스터를 생성하거나 기존 클러스터를 확장하는 과정에서 데이터 컴퓨팅 노드를 추가합니다.

.단계
. System Manager에서 탐색 창에서 * dashboard * 를 선택한 다음 * 상태 * 카드를 선택합니다.
. 추가할 노드가 있는지 확인하고 *세부 정보 보기*를 선택하여 목록을 표시합니다.
+
이 목록에는 아직 AIDE 클러스터에 포함되지 않은 검색된 노드가 포함되어 있습니다.

. 또는 *Cluster*와 *Overview*를 선택한 다음 *Data compute* 탭을 클릭하여 목록을 볼 수도 있습니다.
. 데이터 컴퓨팅 페이지 하단에서 노드 목록 위에 있는 *추가*를 선택합니다.
. *노드 추가* 대화 상자에서 추가할 데이터 컴퓨팅 노드를 선택합니다.
. 노드를 처음 추가하는 경우이고 백엔드 서브넷이 존재하지 않으면 *서브넷 추가*를 선택하고 다음을 제공합니다.
+
** 서브넷 이름(내부 사용)
** 서브넷 주소 및 마스크
** 백엔드 네트워크의 데이터 컴퓨팅 노드 및 ONTAP 노드에 대한 IP 주소 범위
+
System Manager는 해당 범위에 추가되는 모든 데이터 컴퓨팅 노드, 클러스터의 모든 ONTAP 노드, 그리고 데이터 컴퓨팅 노드와 ONTAP 통신에 사용되는 추가 클러스터 수준의 플로팅 IP에 대한 충분한 사용 가능한 IP 주소가 포함되어 있는지 확인합니다.



. 백엔드 서브넷을 추가했는지 또는 이미 존재하는지 여부와 관계없이 다음을 수행하십시오.
+
.. 사용 가능한 IP 주소를 검토하십시오.
.. 필요한 경우 *서브넷 편집*을 선택하고 IP 범위를 확장하십시오.
+
*** 범위는 확장만 가능합니다. 서브넷을 축소하거나 변경하는 것은 지원되지 않습니다.
*** 서브넷 또는 IP 범위를 변경하면 데이터 컴퓨팅 노드에서 기본 Kubernetes 클러스터를 다시 생성해야 할 수 있으며 몇 분 정도 걸릴 수 있습니다.




. 선택적으로 다음을 제공하여 Data Engine 서비스 인터페이스를 구성할 수 있습니다.
+
** 서비스 IP 주소
** 네트워크 마스크
** 게이트웨이(환경에 필요한 경우)
+
단일 IP 주소는 데이터 컴퓨팅 노드 전체에 로드 밸런싱되어 AIDE Console 및 관련 API의 프런트엔드 주소로 사용됩니다.



. 선택한 노드, 백엔드 서브넷 및 Data Engine 서비스 인터페이스 설정을 검토하십시오.
. *추가*를 선택하고 작업이 완료될 때까지 기다리세요. System Manager가 다음 작업을 수행합니다.
+
** 선택한 노드를 데이터 컴퓨팅 클러스터에 추가합니다
** 백엔드 네트워킹을 프로비저닝하고 노드를 Kubernetes 기반 데이터 컴퓨팅 클러스터에 연결합니다.
** 데이터 컴퓨팅 노드 검색을 위한 내부 메타데이터를 업데이트합니다


. 완료되면 *클러스터*와 *개요*를 선택하고 확인하십시오.
+
** *Data compute* 항목 아래에 새로운 Data compute 노드가 클러스터의 일부로 온라인 상태로 표시됩니다.
** 모든 노드가 *정상*입니다
** 각 노드에 대해 AIDE 버전이 올바르게 표시됩니다
** 대시보드 *상태* 카드에 업데이트된 노드 수가 표시되는지 확인하십시오


. (선택 사항) 모든 노드가 클러스터에 참여한 후 모든 서비스가 올바르게 시작되는지 확인합니다.
+
.. Kubernetes Pod 상태 확인:
+
[source, bash]
----
kubectl get pods -n aide-system
----
.. 모든 Pod의 상태가 *Running* 또는 *Completed*로 표시되는지 확인하십시오.
