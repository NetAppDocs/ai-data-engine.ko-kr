= 타사 서버에서 AIDE 소프트웨어 빠른 시작
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
타사 서버에서 AI Data Engine(AIDE) software를 사용하려면 환경을 준비하고, AIDE software를 설치하고, 클러스터를 설정하고, 작업 공간을 생성해야 합니다.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["하나"] AIDE를 새 AFX 클러스터에 설치하시겠습니까, 아니면 기존 AFX 클러스터에 설치하시겠습니까?
[role="quick-margin-para"]
AIDE와 AFX를 동시에 설치할지, 아니면 기존 AFX 클러스터에 AIDE를 통합할지 결정해야 합니다.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["둘"] 타사 서버에서 AIDE 소프트웨어에 대한 요구 사항을 검토하십시오
[role="quick-margin-para"]
시작하기 전에 하드웨어 및 소프트웨어 요구 사항을 검토하십시오. link:../install-aide-server/aide-mde-server-hardware-requirements.html["타사 서버의 AIDE 소프트웨어 요구 사항에 대해 알아보십시오"]. AIDE 소프트웨어를 설치하기 전에 link:../install-aide-server/install-workflow.html["서버를 설정하세요"]해야 합니다.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["3"] 타사 서버에 AIDE 소프트웨어를 설치하십시오
[role="quick-margin-para"]
AFX 스토리지 시스템을 아직 설치하지 않은 경우 https://docs.netapp.com/us-en/ontap-afx/install-setup/install-setup-workflow.html["설치"^] AFX 하드웨어를 설치합니다.

[role="quick-margin-para"]
서버에 RHEL 9.7 및 AIDE 소프트웨어를 설치한 다음 클러스터 네트워크를 구성하십시오. link:../install-setup/install-aide-as-software.html["RHEL 9.7 및 AIDE 소프트웨어를 설치하십시오"].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["네"] 클러스터를 설정하세요
[role="quick-margin-para"]
ONTAP System Manager를 사용하여 AIDE를 AFX 클러스터에 연결하십시오. link:../install-setup/cluster-setup-afx.html["AFX 클러스터를 사용하여 AIDE를 설정합니다"].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["다섯"] 작업 공간 및 데이터 액세스 설정
[role="quick-margin-para"]
link:../workspaces/create-workspace.html["AIDE 데이터에 액세스할 수 있는 작업 공간 및 해당 작업 공간의 사용자를 설정합니다"].

.다음 단계
이제 ONTAP System Manager를 사용하여 AIDE 배포를 관리할 수 있습니다. 작업 공간 권한이 있는 사용자도 link:../metadata-search/explore-metadata.html["AI Data Engine Console에서 Metadata Engine 검색"]을(를) 수행할 수 있습니다.
