= AI Data Engine용 Data-to-RAG 빠른 시작
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
이 워크플로를 사용하여 새로 배포된 AI Data Engine(AIDE) 시스템에서 작동하는 검색 증강 생성(RAG) 엔드포인트로 전환하십시오. 스토리지 관리자, 데이터 엔지니어 및 데이터 과학자가 ONTAP System Manager와 AI Data Engine Console을 사용하여 협업하는 방법을 이해하십시오.


NOTE: 다음 지침은 NetApp DCN 기반 AIDE 배포 환경을 가정합니다.

.시작하기 전에
* NetApp DCN을 설치하고 ONTAP 클러스터에 추가했습니다.
* AIDE를 설치하고 벡터화 및 가드레일 기능을 위한 AIDE 프리미엄 서비스 라이센스를 활성화했습니다.
* link:../install-setup/configure-oidc.html["OpenID Connect(OIDC)"]를 구성하고 관리자, 데이터 엔지니어 및 데이터 과학자 역할에 대한 역할을 매핑했습니다.


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["하나"] 데이터 범위 및 거버넌스를 정의합니다
[role="quick-margin-list"]
스토리지 관리자 또는 보안 관리자로서 AI Data Engine Console과 ONTAP System Manager에서 환경을 준비해야 합니다.

* link:../workspaces/create-workspace.html["하나 이상의 작업 공간 생성"] 로컬 및 원격 데이터 소스에서 가져온 정보입니다.
* link:../guardrails/define-data-policies.html["분류기 및 가드레일 정책 구성"] AI Data Engine Console에서.
* link:../workspaces/assign-users-to-workspace.html["데이터 엔지니어와 데이터 과학자에게 워크스페이스 액세스 권한을 할당합니다"].


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["둘"] 워크스페이스 메타데이터 탐색
[role="quick-margin-list"]
데이터 엔지니어 또는 데이터 과학자로서 AI Data Engine Console을 사용하여 작업 공간 메타데이터를 탐색하려고 합니다.

* link:explore-workspace-metadata.html["워크스페이스 메타데이터 탐색"] 사용 가능한 콘텐츠를 이해합니다.
* RAG에 입력할 하나 이상의 논리적 데이터 하위 집합을 정의합니다(예: 지원 문서, 제품 설명서 또는 익명화된 임상 기록).


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["3"] 데이터 수집 생성 및 게시
[role="quick-margin-list"]
데이터 엔지니어 또는 데이터 과학자로서 선택한 하위 집합을 RAG 지원 컬렉션으로 전환하려고 합니다.

* link:create-data-collections.html["데이터 수집 생성"] 선택한 필터를 사용하여 작업 공간에서 가져옵니다.
* link:create-data-collections.html#publish-a-data-collection["데이터 수집 게시"] 그리고 인덱싱이  `Ready` 상태에 도달할 때까지 모니터링합니다.
* 선택한 컬렉션의 검색 엔드포인트 URI를 복사하여 데이터 과학자 또는 애플리케이션 개발자에게 제공하십시오.
* 필요에 따라 link:view-data-collections.html["데이터 수집 상태 및 벡터 영향 범위를 확인하세요."]
